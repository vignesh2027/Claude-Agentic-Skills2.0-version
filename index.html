⚡ How It Works

Your task. The right agent.
Every time.

AgentOS uses a 4-step orchestration loop to break any task into sub-tasks and dispatch them to specialist agents.

1

Identify

Classifies the domain, complexity, and which agents are needed based on the task.

2

Decompose

Breaks the task into focused sub-tasks, each matched to a specialist's core strength.

3

Route

Dispatches each sub-task to the right agent with the right skill set. Parallel or sequential.

4

Execute

Each agent applies its sub-agents and skills, working at full depth — no hand-waving.

5

Synthesize

All outputs are merged into one unified, coherent response with a clear structure.

6

Deliver

Always includes: Executive Summary · Full Output · Confidence · Next Steps · Caveats.

🚀 Quick Start

Up in 60 seconds.

Four ways to activate AgentOS — pick the one that fits your workflow.

1

Claude.ai (Web)

Create a Project, open Project Instructions, paste the system prompt. Every conversation in that project is now AgentOS.

1. claude.ai → New Project 2. Project Instructions → paste prompt 3. Start chatting
2

Claude Code (CLI)

Append to your global CLAUDE.md so every Claude Code session inherits AgentOS capabilities.

cat prompts/agentOS-system-prompt.md \ >> ~/.claude/CLAUDE.md
3

Anthropic API (Python)

Read the prompt file and pass it as the system message. Works with any Claude model (Sonnet, Haiku, Opus).

with open("prompts/agentOS-system-prompt.md") as f: system = f.read() client.messages.create( model="claude-sonnet-4-6", system=system, messages=[{"role":"user","content":"..."}] )
4

Cursor / Windsurf / IDE

Paste the system prompt into your IDE AI rules or custom instructions file. Works with any Claude-backed IDE extension.

# Cursor: .cursor/rules # Windsurf: .windsurfrules # Paste prompt contents into # the AI rules/instructions file
